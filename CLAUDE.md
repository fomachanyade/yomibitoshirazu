# yomibitoshirazu — agent notes

このファイルは、このプロジェクトで作業するAIエージェント (Claude Code 等) のための
コンテキストとガードレールです。普通の README は別にあります (`README.md` を参照)。

## このプロダクトの芯

縦書きの俳句・短歌・川柳が、壁を流れていく投稿サイト。

- アカウントなし、名前なし、ログインなし
- 一画面に一首ずつ、左から右へゆっくり流れる
- どの詠も等価 (新旧・自他で重み付けしない)
- 完全にステートレス (壁に立つたびに世界はまた新しく始まる)

これは「江戸時代の壁の落書き」のメタファーで、 SNS ではない。
"いいね"、"フォロー"、"通知"、"プロフィール"、"返信" など、人と人を結ぶ機能は
**意図的に存在しない**。詠は誰のものでもなく、宙に流れる。

## 守ってほしいこと (やる前に立ち止まる)

- ユーザー、アカウント、認証、セッション、 cookie の概念を**追加しない**
- 投稿に名前・ハンドル・ID 等の作者情報を**紐づけない**
- "いいね"、"返信"、"シェア"、"通知" 等のソーシャル機能を**追加しない**
- 詠を時系列で並べる UI (タイムライン、新着、ランキング) を**作らない**
- 詠の検索・絞り込み機能を**追加しない**
- ログイン・課金・分析タグ・トラッキングを**追加しない**

これらが必要に見えたら、それは仕様変更の議論であって実装タスクではない。
コミットせず、人間に確認すること。

## 美術方向

- 配色: 浅葱色 (asagi) の壁、墨色は暗いネイビー (`#16213a`)
- フォント: タイトルに EB Garamond、本文に Klee One
- 文字は基本的に縦書き (タイトルの `yomibitoshirazu` だけ横書き)
- ノイズテクスチャは壁の質感を作る重要な要素 (剥がさない)
- 投稿フォームの行動ボタンは「やめる」「流す」(「放つ」ではない)
- 書き込みアイコンは墨の滴 (筆ではない)

## アーキテクチャ

```
ブラウザ ──► Hono (Bun ローカル / Cloudflare Workers 本番)
              ├─ GET  /             → 壁
              ├─ GET  /compose      → 詠む
              ├─ GET  /api/poems/random
              ├─ POST /api/poems
              └─ DB   bun:sqlite (ローカル) / D1 (本番)
```

DB アクセスは `src/db/types.ts` の `PoemRepo` インターフェイスで抽象化している。
**Hono のルート定義 (`src/app.ts`) はランタイム非依存に保つ**こと。
ここに `bun:sqlite` や `D1` を直接 import するとアーキテクチャが崩れる。

## 開発コマンド

```bash
bun install
bun run db:seed   # 種詠を撒く (空のときだけ)
bun run dev       # http://localhost:3000 にホットリロード起動
```

## ディレクトリの意図

```
src/
  server.ts          Bun ランタイム用エントリ。Workers では使わない
  app.ts             Hono のルート定義。ランタイム共通。ここを薄く保つ
  validate.ts        詠の検証
  db/
    types.ts         PoemRepo インターフェイス。実装はここに依存しない
    bun-sqlite.ts    bun:sqlite 実装
    index.ts         ローカルDBファクトリ (server.ts から呼ぶ)
public/              静的ファイル。HTML/CSS/JS を分離して配置
db/schema.sql        DDL の唯一の真実
```

## スタイル

- TypeScript は strict
- コメントは日本語で OK
- 仕様の言葉は和語を尊重 (「投稿」より「詠む」、「送信」より「流す」)
- 過剰なエラーハンドリングや過剰な抽象化は避ける。これは小さな静かなサイト
